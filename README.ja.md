# ASI レース

![ASI レースのゲーム画面](docs/screenshots/gameplay.png)

言語：[EN](README.md) | [JA](README.ja.md) | [FR](README.fr.md) | [ZH-CN](README.zh-CN.md) | [ZH-TW](README.zh-TW.md)

ASI レースは、ブラウザーで動く 3D リアルタイムストラテジーゲームです。AI ラボを思わせる四つの勢力が、計算資源、データ、人材、政府への影響力、社会的信頼、リスク管理、アラインメントをめぐって競争します。最初に ASI 訓練を完了した勢力がゲームを終わらせます。すべてのライバル本部を破壊して勝つこともできます。

> これは非公式のパロディ作品です。OpenAI、Anthropic、Google DeepMind、xAI、またはいかなる実在の研究所とも関係はなく、承認も受けていません。勢力の設定は公開された印象をゲーム向けに抽象化したもので、実在人物は登場しません。

## ソースから実行

このリポジトリには、Fable が生成したパッケージから取り出した共有ブラウザーゲームのソースが入っています。`vendor/three.module.js` も含まれているため、任意のローカル静的サーバーからそのまま実行できます。

```bash
python3 -m http.server 8000
# http://localhost:8000 を開く
```

ヘッドレスのシミュレーション検査は次のコマンドで実行できます。

```bash
npm test
# node test/headless.mjs と node test/bridge.mjs を順に実行
```

## Python 制御インターフェース

依存ゼロの Python SDK（`python/asirace`）から、ゲーム内のあらゆる挙動を制御できます。ヘッドレスでは決定論的シミュレーションをロックステップで進め（スクリプトボット、AI 実験向け）、`?bridge=8765` を付けて開いたブラウザーの対局にはリアルタイムで命令を送り、内蔵 AI から任意の陣営を引き継ぐこともできます。

```python
import sys; sys.path.insert(0, 'python')
from asirace import Game

with Game(seed=42, faction=0) as g:      # Node.js >= 18 が必要
    g.step(seconds=60)
    print(g.observe()['factions'][0]['compute'])
```

サンプルは `python/examples/` にあります（API ツアーの `quickstart.py`、戦場の霧の下で一局を戦い抜く `scripted_bot.py`、観戦・乗っ取り用の `live_control.py`）。設計判断・プロトコル一覧・ベンチマークは [docs/python-bridge.md](docs/python-bridge.md)（英語）/ [docs/python-bridge.zh-CN.md](docs/python-bridge.zh-CN.md)（中国語）を参照してください。`npm run test:py` で Python 側の一気通貫テストを実行できます。

## リリース版

GitHub リリース `v1.1.0` は、`release` GitHub Actions ワークフロー（`packaging/build-zips.sh`）が現在のソースから再構築した二つのプラットフォーム別アーカイブを提供します（内容は [CHANGELOG.md](CHANGELOG.md) を参照）。

- `asi-race-mac-zh.zip`：macOS 版。アプリバンドルと起動スクリプトを含みます。
- `asi-race-win-zh.zip`：Windows 版。コマンド起動ツールと PowerShell のローカルサーバー起動ツールを含みます。

どちらのアーカイブも Three.js エンジンを同梱しているため、解凍するだけで完全オフラインで遊べます。ファイルが失われた場合でも、起動ツールがオンライン起動時に自動で補完します。リリース `v1.0.0` には最初の初版ビルドのアーカイブが残っています。

## 操作

| 入力 | 操作 |
| --- | --- |
| 二本指スワイプ | カメラ移動 |
| ピンチ / ctrl+ホイール | ズーム |
| 二本指タップ / 右クリック | スマート命令：移動、採集、攻撃、集結 |
| クリック / ドラッグ選択 / shift | 選択、範囲選択、追加選択 |
| Q / E、WASD / 方向キー、H | 回転、移動、本部へ戻る |
| [ / ] または alt+ホイール | カメラの俯仰：地平線近くの戦場ショットから鳥瞰まで |
| ctrl+1-4 / 1-4 | 部隊番号の保存 / 呼び出し |
| A + クリック | アタックムーブ：進軍しながら敵を自動で掃討 |
| Tab | 待機中の研究員を順番に選択（カメラも移動） |
| スペース | 直近の襲撃地点へ移動 |
| P、F、M、Esc、? / F1 | 一時停止、二倍速、ミュート、取り消し、マニュアル |

ゲーム内のフィールドマニュアルは五ページ構成で、目標、操作、経済、政治と信頼、エンディングを説明します。

## ゲームプレイ

- 計算資源は本部、データセンター、占領可能な GPU クラスターから得られます。
- データは研究員がマップ上のノードから採集します。高位のラボは合成データを自動生成できます。
- 影響力は議事堂でのロビー活動から生まれ、輸出規制、計算資源補助、規制調査、広報攻勢に使えます。
- 人材はユニット上限を決めます。信頼は採用コストと引き抜きリスクに影響します。
- 研究を急ぐとリスクが増えます。アラインメント研究は事故圧力を下げ、エンディングに影響します。
- 開始時に二つの戦場ライティングを選べます。☀ 昼（暗い虚空に浮かぶ金色の草原と白いキャンパス）と 🌆 夕暮れ（従来のトワイライト）です。
- 映像は映画的な様式化リアリズムを狙っています。フィルミックなカラーグレーディング（彩度低下・ビネット・フィルムグレイン）、実写地表テクスチャと各建設地の本物の切土・盛土整地、トレー状の台座に代わる陣営色コーナー灯柱付きのダークな機械加工デッキ舗装、元のリファレンス映像に合わせて再構築された建物外装（パールホワイト合金パネルとリボンガラス、環境反射付き——素のコンクリートではありません）、人のスケールを伝える外装ディテール（点検はしご・雨樋・吸気グリル）、現実のテックキャンパスを幅広く参照しつつ現実を超えた近未来建築、そして大人のプロポーションのスケルタルアニメーションキャラクターです。
- 動的な天候が戦場を流れます。雲の影が地表を掃き、にわか雨が雨のヴェールで太陽を翳らせ、やがて金色の光が戻ります。昼のグレーディングは晴れやかに調整され、暖色のキーライト、オレンジとティールのスプリットトーン、きらめくハイライトが特徴です。
- 画面下部の HUD はプロフェッショナルな RTS コマンドコンソールです。コーナーブラケットとセンサースイープを備えた戦術マップステーション、陣営ネームプレート・分割式ヘルスバー・生産キューセルを備えたコマンド情報ステーション、ホットキーキャップとコスト表示、ロック/資源不足状態を持つ立体的なコマンドキーのオーダーデッキ——面取りされた三枚の装甲プレートステーションが全幅のシャーシレールに鋲留めされ、全体が選択した陣営のカラーに染まります。
- 地形はゲームシステムです。外周レーンはメサの絶壁で断たれ、軍勢は GPU クラスターの隘路を通るしかありません。斜面は移動を遅らせ、急斜面は通行も建設も拒みます。高所のユニットは視界が伸び、見下ろす攻撃は +30%、見上げる攻撃は −20% です。
- Age of Empires 流の内政レイヤー：各建物で一度きりの経済テクノロジーを研究でき（光インターコネクト、データパイプライン、レッドチーム演習、プロセス監督など）、本部にはスリッページ付きの計算資源スポット市場があります。
- ライバル AI も同じプレイブックで動きます。性格に応じた優先度でテクノロジーを購入し、市場で資源を調整し、あなたの編成を読んでカウンターを训练し、遠方の鉱床へ護衛を付け、消耗した襲撃隊を撤退させ、最後に攻められた方向へ防火壁タワーを増設します。
- 戦場の外では業界のメタ経済が回っています。各ラボの株価と増資、暗号資産バブルや供給不足やオープンソース公開で動くハードウェア指数、クラウド売り出しモード（計算資源を貸してデータと影響力を得つつ、業界全体のハードを安くする）、そしてパロディのスター研究者たち——引退警告、移籍、ヘッドハント、あるいはマップ上でスタートアップを創業し、IPO 前なら誰でも買収できます。
- 終盤は「創発」です。最終訓練は五段階の目覚めであり、各段階の気質はその時点のアラインメント値で決まります——再帰的自己改善が残り時間を溶かし、チェックポイントの商用化が資金とライバルの計算資源転売の値崩れをもたらし、大規模なナラティブ投下（あるいは安全評価の公開）が信頼を曲げ、勝者総取りの人材流入（と内通者の買収）がライバルを枯らし、最後の一手は防御的開示か、合法だが冷酷な電力網容量の買い占めです。二つの訓練が同時に走ると競争的オーバークロックに突入します。
- 戦場は戦争の霧に覆われます。ユニットと建物が視界を確保し、探索済みの地域は最後に見た敵建物だけを残す灰色の記憶に変わります。ミニマップも霧に従い、ASI 訓練の光柱だけは全マップから見えます。
- 勝利は Gen-2、Gen-3、Gen-4 / AGI、ASI 訓練へ進む研究ルート、または全ライバル本部の破壊で達成できます。

プレイ可能な勢力は OpenAI、Anthropic、Google DeepMind、xAI から着想を得ており、それぞれ異なる経済面または安全面のボーナスを持っています。

## プロジェクト構成

```text
index.html            ブラウザーの入口
css/                  HUD、開始画面、戦場オーバーレイのスタイル
js/sim/               DOM や three.js に依存しない決定論的シミュレーション層
js/view/              three.js の表示層、地形、建物、キャラクター、エフェクト
js/ui/                HUD とゲーム内チュートリアル
js/audio/             WebAudio の効果音と環境音楽
js/shared/            シミュレーションと描画で共有する関数
js/bridge/            外部制御プロトコル + ブラウザー実況ブリッジ（?bridge=ポート）
bridge/               ヘッドレスブリッジサーバー（Node、stdio 上の JSON Lines）
python/               Python SDK（asirace）、サンプルボットとスモークテスト
docs/                 スクリーンショットと Python インターフェース技術ノート
test/headless.mjs     Node 用シミュレーション検査
test/bridge.mjs       ブリッジプロトコルとサーバーの検査
vendor/               vendored three.js v0.170.0
packaging/            macOS 版と Windows 版から取り出した起動ツールのソース
```

シミュレーションは固定 0.1 秒ステップで進みます。描画層は状態を読み取り、補間された視覚フィードバックを再生します。AI プレイヤーは人間プレイヤーと同じ命令 API を使います。

## 元のプロンプトの翻訳

```text
ブラウザーで遊べる 3D リアルタイムストラテジーゲームを作ってください。
視点は帝国の時代風の俯瞰視点で、スーパーインテリジェンスへ向かう AI レース
のメタファーにしてください。勢力は OpenAI、Anthropic、Google DeepMind、
xAI で、それぞれブランドから着想を得た個性と、その性格に合うボーナスを
持たせてください。

単なる RTS の見た目替えにはしないでください。このメタファーから仕組みを
発明してください。これらのラボが実際に競っているもの、たとえば計算資源、
データ、人材、政府の支持、世間の見方を考え、それらを経済、技術進行、
勝利条件に変えてください。ライバルのラボは本物の AI 対戦相手としてプレイヤー
と競争し、ゲームは誰かが最初にスーパーインテリジェンスへ到達して終わる
ようにしてください。

Three.js を使い、通常の ES モジュール、ビルド手順なし、ローカル静的サーバー
で動く構成にしてください。すべての素材は実際にダウンロードした高品質素材で
構成し、AI 生成素材は使わないでください。キャラクターはリグ付きモデルで、
スケルタルアニメーションによって実際に歩き、働き、戦う必要があります。
映画的で手応えのある見せ方にしてください。ドラマチックな照明、実際の影、
すべての行動へのフィードバック効果、戦闘、建築、警報の実音効、静かな
背景音楽を含む完全な音風景を用意してください。

操作はトラックパッド優先にし、清潔なオーバーレイ HUD とゲーム内の遊び方
ガイドを入れてください。シミュレーションと描画を分離し、最後にまとめて
ではなく、作りながら各システムをブラウザー上で実際に検証してください。
```

## アセットについて

元のプロンプトは実際にダウンロードしたアセットを要求していました。現在の Fable 出力が実際に依存している外部ファイルは、ダウンロード版の Three.js だけです。建物、キャラクター形状、スケルタルアニメーションのポーズ、テクスチャ、効果音、環境音楽はプロジェクトコードによって手続き的に生成されています。その上で、リポジトリには Poly Haven の CC0 実写地面テクスチャ二組（`assets/textures/`：空撮草地/岩のディフューズとノーマル、乾いた泥のディフューズ）を追加し、地形レンダリングに使用しています。その他の素材は引き続き手続き的に生成されます。このリポジトリはその実装状態を保ち、ソース版をそのまま実行できるよう Three.js を `0.170.0` に固定しています。
