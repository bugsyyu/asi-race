# ASI 競賽

![ASI 競賽遊戲截圖](docs/screenshots/gameplay.png)

語言：[EN](README.md) | [ZH-CN](README.zh-CN.md) | [ZH-TW](README.zh-TW.md) | [JA](README.ja.md) | [FR](README.fr.md)

ASI 競賽是一款在瀏覽器中執行的 3D 即時戰略遊戲。四家 AI 實驗室風格的陣營圍繞算力、資料、人才、政府影響力、公眾信任、風險控制和對齊展開競賽。最先完成 ASI 訓練的一方會結束比賽；摧毀所有對手總部也能獲勝。

> 這是一個非官方戲仿專案。專案與 OpenAI、Anthropic、Google DeepMind、xAI 或任何真實實驗室無關聯、未獲背書；陣營設定是基於公開印象的遊戲化改寫，不包含真實人物。

## 從原始碼執行

本倉庫保存的是從 Fable 生成包中抽出的共享瀏覽器遊戲原始碼，並已經包含 `vendor/three.module.js`，因此原始碼版可以直接透過任意本機靜態伺服器執行。

```bash
python3 -m http.server 8000
# 開啟 http://localhost:8000
```

執行無介面模擬測試：

```bash
npm test
# 等同於 node test/headless.mjs
```

## 發行版下載

GitHub 發行版 `v1.0.0` 包含兩個原始平台壓縮包：

- `asi-race-mac-zh.zip`：macOS 包，包含應用程式包和啟動腳本。
- `asi-race-win-zh.zip`：Windows 包，包含命令啟動器和 PowerShell 本機伺服器啟動器。

如果缺少 `three.module.js`，兩個平台啟動器都會在首次連網啟動時自動下載。下載完成後，遊戲可以離線執行。倉庫原始碼版已經直接包含該檔案。

## 玩法

- 算力來自總部、資料中心和可佔領的 GPU 叢集。
- 資料由研究員從地圖節點採集，高階實驗樓可自動產出合成資料。
- 影響力來自國會遊說，可用於出口管制、算力補貼、監管調查和公關攻勢。
- 人才決定單位上限，信任影響招募成本和被挖角風險。
- 搶跑研發會累積風險，對齊研究可降低事故壓力並影響結局。
- 戰場覆蓋戰爭迷霧：單位與建築提供視野，探索過的區域轉為灰色記憶，敵方建築只保留你最後一次看到的樣子；小地圖同步受迷霧限制，唯有 ASI 訓練的光柱全圖可見。
- 勝利路線包括 Gen-2、Gen-3、Gen-4 / AGI 到 ASI 訓練，也可以摧毀所有對手總部。

可玩陣營受 OpenAI、Anthropic、Google DeepMind 和 xAI 啟發，每一方都有不同的經濟或安全傾向加成。

## 專案結構

```text
index.html            瀏覽器入口
css/                  HUD、開始頁、戰場覆蓋層樣式
js/sim/               確定性模擬層，不依賴 DOM 或 three.js
js/view/              three.js 表現層、地形、建築、角色、特效
js/ui/                HUD 與遊戲內教學
js/audio/             WebAudio 音效和環境配樂
js/shared/            模擬與渲染共用函式
test/headless.mjs     Node 模擬層測試
vendor/               vendored three.js v0.170.0
packaging/            從 macOS 和 Windows 包中抽出的啟動器原始碼
```

模擬層以固定 0.1 秒步長推進。渲染層讀取狀態並播放插值表現效果。AI 玩家與真人玩家使用同一套命令 API。

## 原始提示詞譯文

```text
為我構建一款可遊玩的瀏覽器 3D 即時戰略遊戲，採用類似帝國時代的俯視視角，
把它做成通往超級智慧的 AI 競賽隱喻。陣營包括 OpenAI、Anthropic、
Google DeepMind 和 xAI，每個陣營都要有受品牌啟發的身分，並擁有符合其
個性的加成。

不要只是給即時戰略遊戲換皮；要從這個隱喻中發明機制。思考這些實驗室真正
競爭的東西，例如算力、資料、人才、政府支持和公眾觀感，並把它們變成經濟、
科技進度和勝利條件。對手實驗室應該是真正的 AI 對手，和玩家一起競速；整局
遊戲應以某一方率先抵達超級智慧而結束。

使用 Three.js，採用普通 ES 模組，不需要構建步驟，只依賴本機靜態伺服器。
全部素材都要來自真實下載的高品質素材，不要使用 AI 生成素材。角色應該是
帶骨骼綁定的模型，能夠透過骨骼動畫真正行走、工作和戰鬥。讓它具有電影感
和爽快回饋：戲劇化燈光、真實陰影、每個動作都有回饋效果，並提供完整聲景：
戰鬥、建造和警報都使用真實音效，底下有安靜的背景音樂。

控制方式優先考慮觸控板，覆蓋層 HUD 要乾淨，並提供遊戲內玩法指南。保持模擬
和渲染分離，並在構建過程中即時在瀏覽器中驗證每個系統，而不是最後才驗證。
```

## 資產說明

原始提示詞要求使用真實下載素材。當前 Fable 產出的包實際只依賴下載版 Three.js；建築、角色幾何、骨骼動畫姿態、貼圖、音效和環境音樂均由專案程式碼程序化生成。本倉庫保留這一實作狀態，並把 Three.js 固定為 `0.170.0`，以便原始碼版直接執行。
